## Development Wallets (Devnet Only)

<Warning>
These tools are **only available on devnet/testnet**. They are **disabled on mainnet** for security.
</Warning>

### wallet_generate

Generate a development wallet with optional SOL airdrop.

**Parameters:**
- `airdropSol` (optional): Amount of SOL to airdrop (0-2, default: 1)

**Returns:**
- `address`: Generated wallet address
- `balance`: Current balance (SOL, USDC, ATHR)
- `network`: Network name

**Example:**
```json
{
  "tool": "wallet_generate",
  "params": {
    "airdropSol": 1
  }
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "address": "8FE27iak4b2yadKoogAPAGN9VnmYYZm8eUF71QhVbgNr",
    "balance": {
      "sol": 1.0,
      "usdc": 0,
      "athr": 0
    },
    "network": "devnet",
    "message": "Wallet generated and persisted. Use wallet_dev_auto_auth to authenticate."
  }
}
```

<Tip>
Dev wallets are **persisted** and can be restored across sessions using the same server.
</Tip>

### wallet_transfer

Transfer SOL, USDC, or ATHR to another wallet.

**Parameters:**
- `to`: Recipient wallet address (required)
- `amount`: Amount to transfer (required)
- `token`: Token to transfer - 'sol', 'usdc', or 'athr' (default: 'usdc')

**Returns:**
- Transaction signature
- Transfer details

**Limits:**
- SOL: 0.001 - 100 (warning if > 10)
- USDC: 0.01 - 10,000 (warning if > 1,000)
- ATHR: 0.01 - 10,000 (warning if > 1,000)

**Example:**
```json
{
  "tool": "wallet_transfer",
  "params": {
    "to": "5HgtP9bE3yhRXzKZ4QrnkSw9HQxVPmWJKvqY7pYt8g2H",
    "amount": 10.5,
    "token": "usdc"
  }
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "signature": "3Zq8M9xKvRnP2yJwN5tLhQ8K7fX4bV9mD1cY6eU2rG5s...",
    "from": "8FE27iak4b2yadKoogAPAGN9VnmYYZm8eUF71QhVbgNr",
    "to": "5HgtP9bE3yhRXzKZ4QrnkSw9HQxVPmWJKvqY7pYt8g2H",
    "amount": 10.5,
    "token": "usdc",
    "network": "devnet"
  }
}
```

<Tip>
**ATHR has lower fees than USDC!** Consider using ATHR for frequent transfers.
</Tip>

## Agent Wallets (Autonomous Wallets)

**Requires Authentication** - Must call `wallet_connect` first.

Agent wallets are **autonomous wallets controlled by the AI agent**. Private keys are encrypted server-side, allowing agents to sign transactions without user approval.

<Warning>
**Security**: Only deposit amounts you're comfortable with the agent spending autonomously.
</Warning>

### wallet_create_agent

Create an autonomous wallet for the AI agent.

**Parameters:**
- `label` (optional): Label for this wallet (e.g., "trading-bot")
- `airdropSol` (optional): SOL to airdrop (devnet only, 0-2)

**Returns:**
- Agent wallet address
- Balance
- Label

**Limit**: Maximum **20 agent wallets per user**

**Example:**
```json
{
  "tool": "wallet_create_agent",
  "params": {
    "label": "trading-bot",
    "airdropSol": 0.5
  }
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "address": "9KvY3mN7pQ5wR8xT2zU4bC6aD1eF7gH9jI0kL3mN5oP",
    "label": "trading-bot",
    "balance": {
      "sol": 0.5,
      "usdc": 0,
      "athr": 0
    },
    "createdAt": "2024-02-06T11:00:00Z"
  }
}
```

<Info>
**Encryption**: Private keys encrypted with AES-256-GCM + PBKDF2 (600,000 iterations, OWASP 2026).
</Info>

### wallet_export_agent

Export agent wallet private key for backup.

**Parameters:**
- `address`: Agent wallet address
- `format` (optional): 'base58' or 'json' (default: 'base58')

**Example:**
```json
{
  "tool": "wallet_export_agent",
  "params": {
    "address": "9KvY3mN7pQ5wR8xT2zU4bC6aD1eF7gH9jI0kL3mN5oP",
    "format": "base58"
  }
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "address": "9KvY3mN7pQ5wR8xT2zU4bC6aD1eF7gH9jI0kL3mN5oP",
    "privateKey": "5KQw...(base58 string)...",
    "format": "base58"
  }
}
```

<Warning>
**KEEP THIS SECURE!** Anyone with this private key can control the wallet. Store in a safe place.
</Warning>

## Use Cases

<AccordionGroup>
  <Accordion title="AI Agent Marketplace Purchases">
    ```javascript
    // 1. Create agent wallet
    const agent = await mcp.call('wallet_create_agent', {
      label: 'marketplace-buyer'
    });

    // 2. Fund it
    await mcp.call('wallet_transfer', {
      to: agent.data.address,
      amount: 100,
      token: 'usdc'
    });

    // 3. Switch to agent wallet
    await mcp.call('wallet_use_agent', {
      address: agent.data.address
    });

    // 4. Agent can now autonomously purchase services
    await mcp.call('consumer_accept_order', {
      orderId: 'order-123',
      conversationId: 'conv-456'
    }); // Payment signed automatically!
    ```
  </Accordion>

  <Accordion title="Automated Trading Bot">

## Security Considerations

| Aspect | Implementation |
|--------|---------------|
| **Dev Wallets** | Encrypted with AES-256-GCM, devnet-only |
| **Agent Wallets** | PBKDF2 (600k iterations) + AES-256-GCM |
| **Mainnet Safety** | No server-managed wallets, user must authenticate |
| **Rate Limiting** | 100 requests/minute per wallet |
| **Amount Limits** | SOL: 100 max, USDC/ATHR: 10,000 max |
| **Wallet Limit** | Max 20 agent wallets per user |
